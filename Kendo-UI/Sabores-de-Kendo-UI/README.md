# Sabores de Kendo UI

Descripción | Nivel
------------ | ------------
En este HowTo veremos cuáles sabores del framework Kendo UI están disponibles hoy en día en el mercado. | Principiante

Antes de utilizar Kendo UI es importante conocer qué sabores o "versiones" existen, esto nos ayudará a identificar cuál de ellas se acomoda mejor al proyecto que pensamos desarrollar.

Probablemente ya sabes que Kendo UI es un framework para desarrollo de interfaces gráficas principalmente para aplicaciones web tradicionales así como también para aplicaciones web móviles, haciendo uso de HTML5 Y Javascript. Teniendo esto en cuenta, lo primero que debes saber es que el framework Kendo UI está escrito completamente en Javascript y CSS, esto significa que cualquier tecnología que uses para el desarrollo de tu aplicación web podrá hacer uso de Kendo UI.

Aunque Kendo UI puede ser usado desde páginas/vistas que hagan uso de HTML, Javascript y CSS existen también sabores o implementaciones específicas para integrarse con lenguajes de programación que corren del lado del servidor, estas son concretamente para: ASP.NET MVC, JSP y PHP.

Para entender concretamente estos sabores, veamos cuáles son los nombres formales de cada producto y su tipo de licenciamiento:

Lenguaje Objetivo | Nombre Producto | Tipo Licencia
------------ | ------------ | ------------
HTML/JS | [Kendo UI Professional](http://www.telerik.com/kendo-ui) | Licencia de Pago
HTML/JS | [Kendo UI Core](http://www.telerik.com/kendo-ui/open-source-core) | Gratuito / Open Source
ASP .NET MVC | [UI for ASP.NET MVC](http://www.telerik.com/aspnet-mvc) | Licencia de Pago
JSP | [UI for JSP](http://www.telerik.com/jsp-ui) | Licencia de Pago
PHP | [UI for PHP](http://www.telerik.com/php-ui) | Licencia de Pago

En este momento podrías preguntarte si existe alguna diferencia entre cada versión, así que para responder esa pregunta podemos decir que todas los sabores que requieren un pago son iguales, la única versión que es diferente es la gratuita, su diferencia consta en que tiene menos componentes disponibles.
